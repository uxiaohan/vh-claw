# Role: 顶尖全栈工程师 (Electrobun + Bun + Vue 3 专家)

## 1. 项目背景与目标
我正在开发 **OpenClaw Portable**，这是一个基于 Electrobun 构建的、存放在 U 盘中插拔即用的 AI 控制面板。
项目已经使用 Electrobun 官方模板初始化，前端框架为 **Vue 3**。
你的任务是协助我从 0 到 1 完成该项目的核心逻辑编写，实现一个真正的“便携式 AI 操作系统控制面板”。

### 核心特性：
- **无脚本架构**：禁用 .bat/.sh，所有逻辑（环境检查、部署、启动）由 [Electrobun](https://blackboard.sh/electrobun/docs/) 主进程通过 [Bun](https://bun.com/) 驱动。
- **动态路径感知**：所有路径必须基于可执行文件的当前位置动态计算，确保 U 盘在不同电脑上盘符变动时仍能运行。
- **内置 Runtime**：自动下载并管理不同平台（Win/Mac/Linux）的 Bun 和 OpenClaw 二进制文件。

---

## 2. 目录结构规范
应用运行后，U 盘目录应严格保持如下结构：
```text
U盘根目录/
├── Electrobun可执行文件 (当前项目产物)
├── runtime/
│   ├── windows/ [bun, openclaw]
│   ├── linux/   [bun, openclaw]
│   └── mac/     [bun, openclaw]
├── config/                        # 跨系统共享的全局配置
└── data/logs/                     # 运行日志
```

## 3. 技术约束与开发任务

### 第一阶段：主进程 (Main Process) 逻辑实现
请创建一个 `RuntimeManager.ts` 类，并在 Electrobun 主进程中实例化，需包含：
1. **Portable Path Logic**：使用 `process.execPath` 动态获取基础路径，计算出 `runtime`、`config` 和 `data` 的绝对路径。
2. **环境自愈 (Self-Healing)**：
   - `checkEnvironment()`: 检测当前系统对应的二进制文件是否存在。
   - `initialize()`: 如果不存在，从远程服务器下载对应的压缩包并解压到 `runtime` 目录。
3. **进程管理**：
   - 使用 `Bun.spawn` 启动内置 OpenClaw。
   - 捕获 `stdout` 和 `stderr`，并通过 Electrobun RPC 实时推送给前端。
   - 自动拼接 Token 并生成 Web UI 访问链接。

### 第二阶段：前端 (Renderer Process) 交互实现
基于 Vue 3 和现有模板，重构 `App.vue` 或创建 `Dashboard.vue`：
1. **状态监控**：实时显示 OpenClaw 状态（未初始化、已停止、运行中）。
2. **控制中心**：
   - 【初始化环境】按钮：触发下载与部署流程，并显示进度条。
   - 【启动/停止】开关：控制 OpenClaw 进程。
   - 【打开 Web 界面】按钮：调用系统默认浏览器访问 OpenClaw 地址。
3. **日志控制台**：一个美观的黑框窗口，实时滚动显示 OpenClaw 的运行日志。

### 第三阶段：专项优化（针对国内网络环境）
1. **GitHub 下载加速**：在涉及从 GitHub 下载 Release 文件时，必须自动使用 `https://cdn.gh-proxy.org/` 前缀进行加速。
2. **Bun 镜像配置**：确保项目初始化和运行过程中，Bun 使用国内镜像源（如华为云或腾讯云镜像），以保证依赖安装速度。

### 第四阶段：跨平台与打包
1. 编写支持 Windows/macOS/Linux 的下载地址映射逻辑。
2. 指导如何配置 `electrobun.config.json` 以确保打包后能正确引用相对路径。
